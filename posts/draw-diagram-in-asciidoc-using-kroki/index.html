Krokiを利用してAsciiDocファイルに図を記述する

この記事では，AsciiDocファイルに図を記述し，Krokiを利用して図が埋め込まれたHTMLファイルを生成する手順を説明します。 図の生成にはKrokiのDockerコンテナを使用するため，vscodeのタスク機能を利用して，フォルダを開いたときにコンテナを自動的に起動するよう設定します。

環境

OS

Windows 10 Home 21H2

WSL

Ubuntu 18.04.6 LTS

Docker

Docker Desktop 4.12.0

Visual Studio Code (vscode)

1.73.1

ファイル構成

Cドライブの直下に作成したdocフォルダ内に，この記事で使用するファイルを配置します。 ファイル構成は以下のとおりです。

C:\doc
├── .vscode
│   ├── settings.json
│   └── tasks.json
├── build.sh
├── docker-compose.yml
└── sample.adoc

WSLでフォルダを開く

  1. vscodeを起動します。

  2. F1 → WSL: WSL でフォルダーを開く…​ を選択します。

  3. C:\docを選択します。

vscodeに拡張機能をインストール

vscodeの拡張機能タブから，以下の拡張機能をインストールします。

  • AsciiDoc

  • Docker

AsciiDoc拡張機能を設定

KrokiはPlantUMLやMermaidなど様々な形式のテキストから図を生成するサーバアプリケーションです。

図の生成に使用するKrokiサーバは，インターネットで公開されているものを使用するほか，ローカルに立てて使用することもできます。 ここではインターネットにデータを送信せずに作図するために，ローカルにKrokiサーバを立てます。

Krokiサーバを立てるのは以降の手順で行いますが，AsciiDoc拡張機能がこのKrokiサーバを使用するように，先に.vscode/settings.jsonに以下を追加します。

.vscode/settings.json
{
    // AsciiDocファイルの変換にAsciidoctor.jsを使用する
    "asciidoc.use_asciidoctor_js": true,
    // 図の生成にKrokiを使用する
    "asciidoc.use_kroki": true,
    "asciidoc.preview.attributes": {
        // プレビュー時に図の生成に使用するKrokiサーバのURLを指定する
        "kroki-server-url": "http://localhost:8000"
    }
}

Krokiサーバを起動するDocker Composeファイルを作成

KrokiサーバはDockerコンテナとして実行します。 よって，docker-compose.ymlとして以下を作成します。

なお，このファイルの内容は Using Docker or Podman :: Kroki Documentation の Example of running them all together に記載されているdocker-compose.ymlの内容から blockdiag, bpmn, excalidraw の定義を削除したものです。

docker-compose.yml
version: "3"
services:
  kroki:
    container_name: kroki
    image: yuzutech/kroki
    depends_on:
      - mermaid
    environment:
      - KROKI_MERMAID_HOST=mermaid
    ports:
      - "8000:8000"
  mermaid:
    container_name: kroki-mermaid
    image: yuzutech/kroki-mermaid
    expose:
      - "8002"

vscodeでフォルダを開いたときに自動的にKrokiサーバを起動する

vscodeでAsciiDocファイルを格納したフォルダを開いたときに自動的にKrokiサーバが起動すると便利なので，.vscode/tasks.json[3][4]に以下を記述します。

.vscode/tasks.json
{
    "version": "2.0.0",
    "tasks": [
        {
            // タスクの名前
            "label": "Run docker-compose up",
            // タスクの種類
            "type": "docker-compose",
            // Docker Compose実行時のオプションを指定
            "dockerCompose": {
                "up": {
                    "detached": true
                },
                "files": [
                    "${workspaceFolder}/docker-compose.yml"
                ],
                "projectName": "kroki-server"
            },
            "runOptions": {
                // フォルダを開いたときに自動的に実行する
                "runOn": "folderOpen"
            }
        }
    ]
}

フォルダを閉じて再度フォルダ開きます。 タスクの自動実行を許可するか尋ねられるので，「許可して実行」を選択します。

ここで選択しなかった場合， F1 → タスク: フォルダー内の自動タスクの管理 → フォルダーで自動タスクを許可する を選択しても同様に設定できます。

上記を実施すると，.vscode/settings.jsonに以下が追加されます。

.vscode/settings.json
{
  "task.allowAutomaticTasks": "on"
}

Webブラウザで http://localhost:8000/ にアクセスし，Krokiのページが表示されることを確認します。

AsciiDocファイルを作成

AsciiDocファイルにPlantUMLとMermaidの図を埋め込むには，以下のように記述します。

sample.adoc
= AsciiDocファイルに図を記述するテスト

// 図を出力するディレクトリ
// 図はhtmlファイルに埋め込むので保存する必要がない
:imagesoutdir: .asciidoctor
// 図のキャッシュファイルを配置するディレクトリ
:diagram-cachedir: .asciidoctor

[plantuml]
----
actor アリス
actor ボブ
アリス -> ボブ: こんにちは
----

[mermaid]
----
sequenceDiagram
    actor アリス
    actor ボブ
    アリス->>ボブ: こんにちは
----

AsciiDocファイルのプレビューを表示

デフォルトではKrokiサーバのように外部から取得したデータをプレビューに埋め込むことはできないので，vscodeでsample.adocのタブをアクティブにした状態で， F1 → AsciiDoc: Change Preview Security Settings → Allow insecure local content を選択します。

この状態で画面右上の Open Preview to the Side ボタンをクリックするか， F1 → AsciiDoc: Open Preview to the Side を選択すると，プレビューが表示されます。

AsciiDocファイルをHTMLファイルに変換

build.shとして以下を作成します。

build.sh
#!/bin/bash

docker run --rm -it \
    -v $(pwd):/documents/ \ # (1)
    --network kroki-server_default \ # (2)
    asciidoctor/docker-asciidoctor:1.28 \
    asciidoctor \
        -r asciidoctor-kroki \ # (3)
        -a data-uri -a allow-uri-read \ # (4)
        sample.adoc

  1. カレントディレクトリをコンテナ内の/documentsディレクトリにマウントします。

  2. コンテナをKrokiサーバのDockerコンテナネットワークに接続します。.vscode/tasks.jsonでprojectNameにkroki-serverを指定しているため，kroki-server_defaultというネットワーク名で接続できます。

  3. Krokiで生成した図を使用するために，Asciidoctorの拡張機能であるasciidoctor-krokiを使用します。この拡張機能は本来はAsciidoctorとは別にインストールしておく必要がありますが，asciidoctor/docker-asciidoctorイメージには既にインストールされています。

  4. AsciiDocファイルにdata-uriとallow-uri-readを定義しても反映されないため，asciidoctorコマンド実行時に定義します。

作成したら実行します。

bash build.sh

実行すると以下のsample.htmlが生成されます。
